Simple FieldConstructor Demo

This is a demo on a custom FieldConstructor. The default FieldConstructor renders the label and Input element together. This makes it impossible to render the label and Input element in separate table cells. In the following, a custom FieldConstructor is created to only render the Input element (with the error message if desired). The label is rendered separately by another template. With the new custom FieldConstructor, the label and Input element could be rendered in separate table cells.
In the following, the error of an input field is not displayed next to it. Instead, the field error (only the first one) is displayed at the top. The source code of this demo is here.
